Markdownに記述したローカル画像を自動でAWS S3にアップロードするCLI Markdown-hikoの使い方

yutaka0m Mar 18, 2020 10:56AM

私は普段、Markdownを使ってローカルでブログを書いています。記事を公開する場合、次の手順でやっています。

  1. Markdownで指定していたローカルの画像ファイルを1つ1つサーバ(AWS S3)にアップロードする
  2. Markdownで記述しているローカルの画像パスを、サーバのアドレス(https://xxxx.jpg)に1つ1つ置換していく
  3. 記事をCMSにアップロードして、ブログを公開する

そんな面倒なことをしなくても、ブログサービス(たとえば「はてなブログ」)の機能を使えば、画像をコピー&ペーストするだけでサーバにアップロードしてくれます。

ただし、ブログを引越す時に、画像ファイルがブログサービスに依存していると大変です。画像を引越し先のブログサービスに転送して、画像のURLを1つ1つ置換して…と途方もないい時間がかかります。
なので、画像はブログサービスとは依存しない方法で配信したいという持ちベージョンがあります。

やったこと

画像をサーバにアップロードして、画像パスをサーバのURLに置換する作業は、とても非生産的です。なので、この作業を自動化しました。

  1. Markdownで指定したローカルの画像ファイルを
  2. 自動的にAWS S3にアップロードして
  3. 自動的に画像のパスをサーバのURLに置換

名前はMarkdown-hikoといいます。Pythonで書いたCLIです。

Markdown hiko の使い方

Markdown-hikoの使い方を説明します。画像配信用のAWS S3を立てるところからやります。

AWS S3のバケットをつくる

S3のコンソール画面にいきます。

https://console.aws.amazon.com/s3/home

バケットを作成を選択して次のように入力します。

  • バケット名: 任意の名前
  • リージョン: 日本ならap-northeast-1を選びます
  • パブリックアクセスをすべてブロックのチェクを外します

最後に、現在の設定により、このバケットと中のオブジェクトがパブリックになる可能性があることを了承します。のチェックをつけて、バケットを作成をクリックします。

バケット一覧から、作成したバケットをクリックします。プロパティ -> Static website hostingをクリックします。

次のように、このバケットを使用してウェブサイトをホストするにチェックを入れて、インデックスドキュメントにindex.htmlと入力します。 このときに、表示されているエンドポイントをメモしておきます。

続いて、アクセス権限 -> パケットポリシーに次のコードを記入します。

xxxxxのところにはバケット名を入力します。

{
  "Version":"2012-10-17",
  "Statement":[
    {
      "Sid":"AddPerm",
      "Effect":"Allow",
      "Principal": "*",
      "Action":["s3:GetObject"],
      "Resource":["arn:aws:s3:::xxxxx/*"]
    }
  ]
}

この設定をすると、バケット内がすべてアクセス可能になるので、セキュアな情報を保存する場合はお気をつけください。

以上で、画像を保存するS3の設定は完了です。S3の前にCloudFrontを立てて、独自ドメインを指定した方がよりよいと思います。参考：CloudFront 経由で S3 のファイルにアクセスする – Qiita

Markdown-hikoを使う

インストール

S3の設定が終わったので、Markdown-hikoの使い方を説明します。

まずは、Mardown-hikoをインストールします。

$ pip install --user markdown-hiko

※ markdown-hikoが実行できない場合はPATHを通してください。Macの場合、$HOME/.local/binがデフォルトのインストールパスです。

Markdownを書く

たとえば、次のようなディレクトリ構造だったとします。

/tmp
 └── markdown
      └── doc
      │    └── test.md
      └── img
           └── test-image.jpg

test.mdから./img/test-image.jpgの画像を表示するには、

例1:

# Test画像 例1
![](../img/test-image.jpg)

と記述します。

また、絶対パスで記述したり、

例2:

# Test画像 例2
![](/tmp/markdown/img/test-image.jpg)

プロジェクトルートが/tmp/markdownだとすると、

例3:

# Test画像 例3
![](/markdown/mg/test-image.jpg)

と書くこともできます。 （GitHubで画像を表示する際は、プロジェクトルートからのパスなので、この指定方法をよく使います）

Markdown-hikoを使う

例1と例2で、Markdown-hikoを使用する場合を説明します。

先ほど作成したS3のホスティングURLがhttps://markdown-hiko-test-bucket.s3-ap-northeast-1.amazonaws.comで、バケット名がmarkdown-hiko-test-bucketだったとすると、次のようのなります。

$ export AWS_ACCESS_KEY_ID=xxxxx #AWSのアクセスキーを入力
$ export AWS_SECRET_ACCESS_KEY=xxxxx #AWSのシークレットアクセスキーを入力
$ export AWS_DEFAULT_REGION=ap-northeast-1 #AWSのデフォルトリージョンを入力
$ cd /tmp/markdown/doc
$ markdown-hiko \
      -f ./test.md \
      -u "https://markdown-hiko-test-bucket.s3-ap-northeast-1.amazonaws.com" \
      -b "markdown-hiko-test-bucket"

これは例なので、実際に使用するときは、-f, -u, -bに指定する値は、自分の環境のものを使用してください。

AWSのアクセスキーがわからない場合は、IAM ユーザーのアクセスキーの管理の手順で発行してください。

コマンドを実行すると、Markdownファイルは次のように変換されているはずです。画像は自動で指定したS3のバケットにアップロードされています。

# Test画像 例1
![](https://markdown-hiko-test-bucket.s3-ap-northeast-1.amazonaws.com/test-image.jpg)

また、例3の書き方でmarkdown-hikoを使用する場合、プロジェクトルートを-rオプションで指定してあげる必要があります。

$ export AWS_ACCESS_KEY_ID=xxxxx #AWSのアクセスキーを入力
$ export AWS_SECRET_ACCESS_KEY=xxxxx #AWSのシークレットアクセスキーを入力
$ export AWS_DEFAULT_REGION=ap-northeast-1 #AWSのデフォルトリージョンを入力
$ cd /tmp/markdown/doc
$ markdown-hiko \
      -f ./test.md \
      -u "https://markdown-hiko-test-bucket.s3-ap-northeast-1.amazonaws.com" \
      -b "markdown-hiko-test-bucket" \
      -r /tmp/markdown
