Terraformでlambdaを実行する

yutaka0m Mar 16, 2020 6:08AM

Terraformを使って、AWS Lambdaの関数定義を作成します。

Lambdaのソースコードをコピー&ペーストしたり、手動でzip化したりするのが面倒なので、ソースコードもTerraformの機能を使ってzip化してアップロードします。

ディレクトリ構造

次のようなディレクトリ構成にします。

.
├── config.tf
├── main.tf
├── output
│   └──
└── src
    └── lambda_function.py

/outputは、/srcをzipにしたファイルが吐き出されるフォルダなので、空のフォルダを作成しておきます。

各ファイルの説明

config.tfには、Terraformを実行するために必要な情報を記述しています。

provider "aws" {
  region     = "ap-northeast-1"
}
terraform {
  required_version = ">= 0.12"
}

lambda_function.pyは、サンプルのlambda関数です。実際には、ここにコードを実装してきます。

import json
def lambda_handler(event, context):
    return {
        'statusCode': 200,
        'body': json.dumps('Hello from Lambda!')
    }

main.tfで、lambda関数を定義しています。

data "archive_file" "sample_function" {
  type        = "zip"
  source_dir  = "./src"
  output_path = "./output/lambda_function.zip"
}
resource "aws_lambda_function" "test_function" {
  filename         = data.archive_file.sample_function.output_path
  function_name    = "test_function"
  role             = aws_iam_role.iam_for_lambda.arn
  handler          = "lambda_function.lambda_handler"
  source_code_hash = data.archive_file.sample_function.output_base64sha256
  runtime          = "python3.8"
  memory_size = 128 # 最適な値にしてください
  timeout     = 30 # 最適な値にしてください
}
resource "aws_iam_role" "iam_for_lambda" {
  name = "iam_for_lambda"
  assume_role_policy = <<EOF
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Action": "sts:AssumeRole",
      "Principal": {
        "Service": "lambda.amazonaws.com"
      },
      "Effect": "Allow",
      "Sid": ""
    }
  ]
}
EOF
}

archive_fileでsrcフォルダをzip化しています。
また、aws_lambda_functionでAWSのLambda定義をしています。

実行

コマンドラインで次のようなコマンドを実行します。

terraform init
terraform apply
# 変更内容が表示されるので、「yes」と入力

AWSのコンソールを確認すると、lambda関数が作成されています。

関数の中身を確認すると、ローカルで書いた関数が反映されています。

あとは、ソースコードを編集するごとにterraform applyをすればAWSに反映されます。
