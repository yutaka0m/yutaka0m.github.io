GItHub Actions プライベートリポジトリに複数のアクションを同居させる

yutaka0m Feb 19, 2020 9:00AM

GitHub Actionsを利用していると、「プライベートな共通の処理」をどこかにまとめておきたくなります。

アクションを作成するには、JavaScript アクションを作成するを参考にすると良いのですが、1リポジトリ1アクションの前提で説明されています。

この手法だと、共通の処理を1つ作るごとにリポジトリを作成するため、リポジトリが乱立してしまいます。

そこで、1つのリポジトリに複数のアクションを同居できないか検証をしてみました。

実装

ディレクトリ 構造

適当な名前のリポジトリを作成して、以下のようなディレクトリー 構造にしました。

リポジトリ名 : PrivateActions

GoodbyeAction/
├—action.yml
├—index.js
└—package.json
HelloAction/
├—action.yml
├—index.js
└—package.json
.github/
└—workflows/
  ├—goodbye.yml
  └—hello.yml

GoodbyeActionとHelloActionそれぞれのフォルダーの配下にアクション定義が記述されています。

コード

GoodbyeAction/action.yml

name: ‘Goodbye World’
description: ‘Sey Goodbye’
runs:
  using: ‘node12’
  main: ‘index.js’

GoodbyeAction/index.js

async function run() {
    console.log(“Goodbye”)
}
run()

HelloAction/action.yml

name: ‘Hello World’
description: ‘Sey Hello’
runs:
  using: ‘node12’
  main: ‘index.js’

HelloAction/index.js

async function run() {
    console.log(“hello”)
}
run()

ワークフロー

JavaScript アクションを作成するにも書かれていますが、プライベートリポジトリのアクションを実行するには、チェックアウトする必要があります。

チェックアウトしたうえで、それぞれのアクションのフォルダーの相対パスをuses: で指定すると実行できます。

goodbye.yml


name: GOODBYE
on: [push]
jobs:
  build:
    runs-on: ubuntu-latest
    steps:
    - uses: actions/checkout@v2
    - name: Run GoodbyeAction
      uses: ./GoodbyeAction

hello.yml

name: HELLO
on: [push]
jobs:
  build:
    runs-on: ubuntu-latest
    steps:
    - uses: actions/checkout@v2
    - name: Run HelloAction
      uses: ./HelloAction

他のリポジトリから利用するテスト

別のリポジトリを作成して、作成したプライベートリポジトリのアクションを呼び出すことができるかテストしてみます。

トークン発行

外部のプライベートリポジトリを参照するにはトークンが必要になります。

GitHub -> Settings -> Personal access tokens で「Generate new token」をクリックします。

Noteに適当な名前を入れて、repoに☑️します。

最下のGenerate tokenをクリックするとトークンが表示されるので、コピーしておきます。

トークンをSecretsに登録する

適当な名前のリポジトリを作成して、

Settings -> Secrets -> Add a new secret をクリックします。

Nameに適当な名前を 入れて、Valueのところにトークンをペースト。Add secretをクリックして保存します。

ワークフロー

以下のようにワークフローを作成します

.github/workflows/test.yml

name: TEST
on: [push]
jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout PrivateActions
        uses: actions/checkout@v2
        with:
          repository: yutaka0m/PrivateActions
          path: PrivateActions
          token: ${{ secrets.ACTIONS_ACCESS_TOKEN }}
      - name: Run GoodbyeAction
        uses: ./PrivateActions/GoodbyeAction
      - name: Run HelloAction
        uses: ./PrivateActions/HelloAction

結果

それぞれのアクションが実行できていることを確認できました。

まとめ

「1つのリポジトリに複数のアクションを同居できる」ということがわかりました。

これでアクション用のリポジトリが量産されることは防げそうです。
